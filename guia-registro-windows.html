CONCEITO

O que é HKCU\Environment?

É a chave do Registro do Windows que armazena as variáveis de ambiente do usuário. Diferente de variáveis de sessão (que morrem quando o terminal fecha), variáveis no registro persistem entre sessões — incluindo reboots. Tanto PowerShell quanto CMD (setx) escrevem nestas mesmas chaves.

HKEY_CURRENT_USER\Environment
Caminho no regedit

Os três escopos

Cada escopo tem propósito, localização e nível de persistência diferentes:

Escopo Localização Persiste? Admin? PowerShell CMD
Process
Só este terminal		 Memória (RAM) Não Não $env:VAR = "valor" set VAR=valor
User
Seu usuário		 HKCU\Environment Sim Não [Environment]::SetEnvironmentVariable("VAR", "valor", "User") setx VAR "valor"
Machine
Todos os usuários		 HKLM\...\Environment Sim Sim [Environment]::SetEnvironmentVariable("VAR", "valor", "Machine") setx VAR "valor" /m

🧠 Hierarquia de resolução

Quando um processo lê $env:VAR, o Windows busca nesta ordem:

Process → User (HKCU) → Machine (HKLM) → System
COMANDOS

Criar, ler, listar e deletar

1

Criar / Atualizar

[Environment]::SetEnvironmentVariable(
"VAR", "valor", "User")
CMD setx VAR "valor"
CMD setx VAR "valor" /m admin

⚠️ Importante: variáveis não aparecem na sessão atual — abra um novo terminal.

2

Ler

$env:VAR
Get-ItemPropertyValue
-Path "HKCU:\Environment"
-Name "VAR"
CMD echo %VAR%
CMD reg query HKCU\Environment /v VAR
3

Listar todas

Get-ItemProperty -Path "HKCU:\Environment"
CMD reg query HKCU\Environment
4

Deletar

[Environment]::SetEnvironmentVariable("VAR", $null, "User")
Remove-ItemProperty -Path "HKCU:\Environment" -Name "VAR"
CMD reg delete HKCU\Environment /v VAR /f

🔄 Recarregar sem abrir novo terminal

foreach ($level in "Machine","User") {
[Environment]::GetEnvironmentVariables($level).GetEnumerator() | % {
Set-Item -Path "Env:\$($_.Name)" -Value $_.Value
}
}

⚠️ setx: limite de 1024 chars

Trunca valores longos silenciosamente. Evite para PATH — use PowerShell.

⚠️ setx: %VAR% expandido

setx PATH %PATH%;... congela referências como %JAVA_HOME%. Use PowerShell para dinamismo.

BOAS PRÁTICAS

Como usar bem

📛

Nomenclatura

MAIÚSCULAS com underscores. Prefixo do projeto/serviço.

OPENROUTER_API_KEY
GITHUB_TOKEN
LOG_LEVEL=debug
🔒

Segurança

Em máquina pessoal, env var User é suficiente para chaves de API. Para senhas reais, prefira Credential Manager (cmdkey).

🧹

Manutenção

Mantenha uma lista do que você criou. Documente em um SETUP.md. Limpe variáveis que não usa mais.

ARMAZENAMENTO

Métodos de armazenamento de configs

🔐 Credential Manager (cmdkey)

Se preferir segurança criptografada, o Credential Manager do Windows é uma alternativa ao registro:

Salvar
cmdkey /generic:OPENROUTER_API_KEY /user:token /pass:"sk-or-v1-..."
Listar
cmdkey /list
FAQ

Solução de Problemas

"SetEnvironmentVariable não funciona" +

Se estiver usando "Machine", precisa executar o PowerShell como Administrador. Para "User", qualquer terminal funciona.

"Criei mas $env:VAR continua vazia" +

Comportamento esperado. Variáveis User só carregam em sessões novas. Solução: abra um novo terminal ou use o script de recarregamento da seção "Carregar na sessão atual".

"Como confirmar que foi salva?" +

Use Get-ItemProperty -Path "HKCU:\Environment" -Name "MINHA_VAR" -ErrorAction SilentlyContinue. Se retornar o valor → salva. Se retornar erro → não existe.

"Variável aparece no registro mas não no terminal" +

O Windows notifica processos via WM_SETTINGCHANGE. Se não funcionou, force com [Environment]::SetEnvironmentVariable("VAR", "valor", "User") ou reinicie o Explorer: Stop-Process -Name explorer -Force.

"Quero remover uma variável que criei" +

PowerShell: Remove-ItemProperty -Path "HKCU:\Environment" -Name "VAR_INDESEJADA"

CMD: reg delete HKCU\Environment /v VAR_INDESEJADA /f

⚠️ setx VAR "" não remove — apenas zera o valor, mas a entrada no registro continua lá.

"Qual a diferença entre setx e PowerShell?" +

Os dois escrevem no mesmo lugar do registro (HKCU\Environment). A diferença prática:

  • setx: funciona em CMD e scripts .bat, mas tem limite de 1024 caracteres e expande variáveis (%PATH% vira texto fixo)
  • PowerShell: sem limite prático, não expande referências, ideal para PATH e valores longos
